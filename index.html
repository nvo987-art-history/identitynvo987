Overview of the identity.nvo987.us Infrastructure

The https://identity.nvo987.us domain serves as the decentralized identity node of the NVO987 research infrastructure. This system provides the official publication of the did:web:identity.nvo987.us decentralized identifier, along with the associated metadata, cryptographic verification information, and infrastructure documentation.

The purpose of the identity node is to provide a publicly verifiable digital identity reference point for research systems operating on the web, federated services, and decentralized identity infrastructures.

The documents published on this site enable external systems to automatically discover, resolve, and verify the identity, as well as the infrastructure components associated with it.

Decentralized Identifier (DID)

The central element of the infrastructure is the did:web:identity.nvo987.us identifier. This identifier is a web-based identity built on the W3C Decentralized Identifiers standard, defining a domain-bound, publicly verifiable digital identifier.

The DID document is available at the following location:

https://identity.nvo987.us/.well-known/did.json

The document contains the verification keys, authentication methods, and service endpoints associated with the identity.

The relationship between the domain and the decentralized identifier is defined in the following configuration document:

https://identity.nvo987.us/.well-known/did-configuration.json

This document enables external systems to verify that the domain owner controls the decentralized identifier.

Identity Discovery and Metadata Endpoints

The identity node publishes several standard discovery and infrastructure metadata documents that enable automatic identity resolution across federated and decentralized systems.

  • https://identity.nvo987.us/.well-known/webfinger – WebFinger identity discovery endpoint
  • https://identity.nvo987.us/.well-known/nodeinfo – infrastructure metadata document
  • https://identity.nvo987.us/.well-known/openid-configuration – OpenID Connect discovery configuration
  • https://identity.nvo987.us/.well-known/security.txt – security contact information

These documents allow identity providers, federated networks, and automated systems to recognize and integrate the identity.

Cryptographic Verification

The authenticity of the identity is based on public cryptographic keys.

The primary public key is available at:

https://identity.nvo987.us/keys/identity-public.pem

Metadata about the key can be found in the following document:

https://identity.nvo987.us/keys/identity-key.json

These files enable external systems to verify digital signatures and documents associated with the identity.

Digitally Signed Documents

The infrastructure publishes several digitally signed documents that cryptographically confirm statements related to the identity and the research infrastructure.

  • https://identity.nvo987.us/signed/identity-statement.json – identity declaration
  • https://identity.nvo987.us/signed/research-statement.json – description of research activity
  • https://identity.nvo987.us/signed/knowledge-graph-signature.json – knowledge graph integrity signature
